# System Design for AI Systems

AI systems are distributed systems first. This module covers the system design fundamentals that every AI architect must know.

## 🚀 Roadmap & Sequence

### 1. Distributed Systems Foundations
*   **01. Distributed Systems Fundamentals:** CAP theorem, consistency models, replication strategies, distributed transactions.
*   **02. API Design:** REST principles, versioning, pagination, rate limiting, gRPC vs REST vs GraphQL.

### 2. Data Infrastructure
*   **03. Message Queues:** Kafka, RabbitMQ, SQS — topics, consumer groups, DLQs, backpressure.
*   **04. Database Selection:** Relational vs NoSQL vs Vector DB, indexing, sharding, connection pooling.

### 3. AI-Specific System Design
*   **05. ML System Design:** Feature stores, model registry, serving patterns, drift detection, champion-challenger.
*   **06. Scalability Patterns:** Load balancing, consistent hashing, caching hierarchy, CDN for AI.
